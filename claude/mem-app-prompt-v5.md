# MEM (My Enneagram Map) 앱 제작 프롬프트 v5

## 📋 프로젝트 개요

에니어그램 기반 통합 성장 플랫폼 "MEM"을 만들어주세요. 이 앱은 **27문항 에니어그램 검사(1~10점 척도)**를 통해 사용자의 성격 유형을 정밀 분석하고, 학습 전략, 진로 매칭, 대인관계 전략을 제공하는 모바일 친화적 웹앱입니다.

**핵심 특징:**
- 1~10점 세밀한 척도로 정밀 측정
- 에니어그램을 모르는 사람도 직관적으로 이해할 수 있는 UI
- 36가지 유형 조합별 관계 설명

---

## 🎨 디자인 요구사항

### 전체 스타일
- **컬러 테마**: 다크 모드 기반, 그라데이션 배경 (indigo-purple-pink 계열)
- **UI 스타일**: 글래스모피즘 (backdrop-blur, 반투명 배경)
- **모서리**: 둥근 모서리 (rounded-2xl, rounded-3xl)
- **반응형**: 모바일 퍼스트 (max-width 제한)

### 컴포넌트 스타일
```
카드: bg-white/10 backdrop-blur-lg rounded-2xl border border-white/20
버튼 (주요): bg-gradient-to-r from-pink-500 to-purple-600 text-white rounded-xl
버튼 (보조): bg-white/10 border border-white/10 text-white
입력 필드: bg-white/10 rounded-xl text-white placeholder-white/40 border border-white/10
프로그레스 바: bg-gradient-to-r from-pink-400 to-purple-400
```

## 🗂️ 화면 구조

### 1. 스플래시 화면 (splash)
```
- 앱 로고 (🧭 이모지 + 글래스 배경)
- 앱 이름: "MEM 3.0"
- 부제: "통합 성장 플랫폼"
- 태그라인: "학습 · 진로 · 관계 · 성숙"
- "27문항 정밀 에니어그램" 표시
- 로그인 버튼 / 시작하기 버튼
```

### 2. 인증 화면 (auth)
```
- 뒤로가기 버튼
- 로고 + 환영 메시지
- 회원가입: 이름, 이메일, 비밀번호
- 로그인: 이메일, 비밀번호
- 모드 전환 링크
```

### 3. 웰컴 화면 (welcome)
```
- ✨ 이모지
- "나를 알아가는 여정" 제목
- "27문항 정밀 에니어그램 검사" 설명
- "1~10점 세밀한 척도로 정밀 측정" 안내
- 예상 소요시간: 5~7분
- 테스트 시작 버튼
```

### 4. 검사 화면 (survey) - ⭐ 1~10점 척도
```
헤더:
- 테스트 제목 + 진행률 (n/27)
- 프로그레스 바 (애니메이션)
- 응답 완료 개수

문항 영역:
- 문항 텍스트 (중앙 정렬, 큰 글씨)

점수 표시 영역:
- 현재 선택 점수 (큰 숫자)
- 점수 라벨 (전혀 아니다 ~ 매우 그렇다)
- 점수별 색상 그라데이션

점수 선택 UI:
- 1~10 버튼 그리드 (10열)
- 선택된 버튼 강조 (scale-110, shadow-lg)
- 양쪽 라벨: "전혀 아니다" / "매우 그렇다"

슬라이더:
- range input (1~10)
- 커스텀 스타일 thumb

네비게이션:
- 이전 버튼 (첫 문항 제외)
- 다음 버튼 (점수 선택 후 활성화)
- 마지막 문항: "결과 보기" 버튼
```

### 5. 결과 화면 (result)
```
- 🎉 축하 이모지 + "분석 완료!"
- 핵심 유형 카드:
  - 유형 이모지 (대형)
  - 유형 번호 + 이름
  - 유형 키워드
  - 날개 유형
  - 유형 설명
- 상위 3개 유형 점수 바 차트 (이모지 포함)
- 성장/스트레스 방향 2컬럼 카드 (이모지 + 유형명)
- 대시보드 이동 버튼
```

### 6. 대시보드 (dashboard)
```
헤더:
- 유형 이모지 + 사용자 이름
- 유형 정보 (n번 유형명 · 날개n)
- 연속 기록 배지 (🔥 n일)
- 로그아웃 버튼

하단 탭 네비게이션:
- 🏠 홈 / 📚 학습 / 🧭 진로 / 💝 관계
```

### 점수 계산 로직 (10점 척도용)
```javascript
const calculateProfile = (answers, questions) => {
  // 1. 유형별 점수 합산
  const typeScores = {};
  for (let t = 1; t <= 9; t++) typeScores[t] = 0;
  
  questions.forEach((q, idx) => {
    if (answers[idx] !== undefined) {
      typeScores[q.type] += answers[idx];
    }
  });

  // 2. 백분율 변환 (각 유형 최대 30점 = 10점 × 3문항)
  const maxScore = 30;
  const typePercentages = {};
  for (let t = 1; t <= 9; t++) {
    typePercentages[t] = (typeScores[t] / maxScore) * 100;
  }

  // 3. 핵심 유형 결정
  const sorted = Object.entries(typePercentages).sort((a, b) => b[1] - a[1]);
  const coreType = parseInt(sorted[0][0]);
  const secondType = parseInt(sorted[1][0]);

  // 4. 날개 결정 (인접 유형 중 높은 점수)
  const leftWing = coreType === 1 ? 9 : coreType - 1;
  const rightWing = coreType === 9 ? 1 : coreType + 1;
  const wingType = typePercentages[leftWing] > typePercentages[rightWing] ? leftWing : rightWing;

  return { 
    core_type: coreType, 
    wing_type: wingType,
    second_type: secondType,
    all_scores: typePercentages,
    raw_scores: typeScores,
    integration: integrationPaths[coreType]
  };
};
```

### 리스트형 호환성 카드
기존 3x3 그리드 대신 세로 리스트로 변경하여 정보를 더 명확하게 표시합니다.

**각 항목에 표시할 정보:**
- 유형 이모지
- 유형 번호 + 이름
- 유형 키워드 (예: "원칙과 정확함")
- 호환도 퍼센트 + 레이블 (최고/좋음/보통)
- 관계 설명 미리보기

**호환도별 스타일:**
- 85%+ 최고: 초록색 (text-emerald-400)
- 70%+ 좋음: 노란색 (text-yellow-400)
- 그 외 보통: 회색 (text-slate-400)
